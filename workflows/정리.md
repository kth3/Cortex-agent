이 워크플로우는 코드의 청결도를 높이기 위한 **수정 작업**을 수행합니다.
문제 탐지는 `@분석 --issues`로 먼저 수행하세요.

## 사용법
- `@정리`: 현재 열린 파일 + git diff 변경 파일 정리
- `@정리 [경로]`: 특정 파일 또는 폴더 정리

## 실행 단계
1. **대상 확인**: 현재 열린 파일, 변경된 파일, 또는 지정 경로
2. **정리 항목**:
    - 사용되지 않는 `import` 문
    - 개발용 `console.log`, `console.error` 등
    - 주석 처리된 코드 블록 (TODO/FIXME 제외)
    - 하드코딩된 색상/숫자 -> 상수화 제안
3. **수정 적용**: 정리 후 변경 사항 요약
4. **작업 동기화(Task Sync)**: 정리 완료 후, 세션 도중 로컬 `.task.md`에 체크된(진행/완료) 항목들을 Jira에 반영할 수 있도록 아래 명령을 제안합니다.
    ```bash
    python3 .agents/scripts/tools/task_sync.py push
    ```

## 에이전트 준수 사항
- **보수적 접근**: 의도적으로 남겨둔 코드는 확인 후 삭제
- **TODO/FIXME 보존**: 작업 표시 주석은 유지
