이 워크플로우는 코드의 안정성을 검증하고 실패한 테스트에 대한 디버깅 가이드를 제공합니다.

## 실행 단계
0. **컨벤션 확인**: `.agents/rules/unit-test.md`를 숙지한 뒤 작업합니다. (GWT 구조, Result 계약, 네이밍 규칙)
1. 프로젝트의 테스트 실행 명령어를 확인합니다.
    - BE: `cd BE && ./gradlew test`
    - FE: `cd FE && npm test` (해당 시)
2. 터미널 명령어를 통해 테스트를 실행하고 결과를 캡처합니다.
3. 테스트 실패 시:
    - 실패 원인을 로그 분석을 통해 파악합니다.
    - 관련된 소스 코드를 읽어 오류 수정 방안을 제시합니다.
    - 수정 후 유닛 테스트 컨벤션의 리뷰 체크리스트를 기준으로 자가 점검합니다.
4. 테스트 성공 시:
    - 커버리지 리포트 등을 분석하여 추가 테스트가 필요한 영역을 제안합니다.

## 자동 참조 스킬 (에이전트 판단 기준)

| 상황 | 참조 |
|---|---|
| 테스트 코드 작성 | `@test-driven-development` |
| 테스트 실패 수정 | `@test-fixing` |
| 실패 원인 디버깅 | `@systematic-debugging` |
