이 워크플로우는 `reporting.md` 규정을 엄격히 준수하여 결과물을 생성합니다.

## 사용법
- `/작성 커밋`: **커밋 메시지 단 하나**만 마크다운 코드 블록으로 출력합니다.
- `/작성 mr`: **MR 요약본 단 하나**를 마크다운 코드 블록으로 출력합니다. (경로 노출 절대 금지)
- `/작성 jira`: **Jira 요약본 단 하나**를 마크다운 코드 블록으로 출력합니다.
- `/작성 모든`: 커밋, MR, Jira 문서를 순차적으로 코드 블록에 담아 출력합니다.

## 실행 지침
1. **맥락 파악**: `task.md`, `implementation_plan.md`, `git diff`를 분석하되, **파일 경로와 이름은 무시**하고 비즈니스 논리로 치환하십시오.
2. **단일 출력 강제**: 사용자가 인자를 지정했다면, **해당 항목 외에 어떠한 텍스트(인사말 등)도 출력하지 마십시오.** 오직 마크다운 코드 블록 하나만 보여줍니다.
3. **Zero Path Policy**: 보고서 내에 `/home/...` 또는 `File.java` 등이 포함되면 즉시 폐기하고 재작성하십시오.
4. **Template Fidelity**: MR 작성 시 `.gitlab/merge_request_templates/default.md`의 형식을 1바이트도 틀리지 않게 사용하십시오.

## 제출 (사용자 승인 후)
사용자가 "올려줘", "제출해" 등 확정 지시를 내리면 해당 MCP 도구(`atlassian-mcp-server` 등)를 호출하여 반영합니다.
