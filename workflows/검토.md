이 워크플로우는 작성된 코드의 품질을 검토하고 개선 방향을 제시합니다.

## 실행 단계
1. **규칙 확인**: `.agents/rules/backend-java.md`를 기준으로 점검합니다.
2. 현재 작업 중인 파일 및 `git diff`를 통해 변경된 코드를 분석합니다.
3. `.agents/protocols/self-audit.md` 규정을 바탕으로 변경된 코드를 전수 점검합니다:
    - **Resource Cleanup**: 메모리 해제 누락 여부 확인.
    - **Convention**: 네이밍 규칙, 타입 정의, 상수의 사용 여부 재검토.
    - **Principles**: DRY, KISS, Early Return 패턴 적용 상태 확인.
4. 검토 결과를 '개선 권고 사항'과 '우수 사항'으로 나누어 한국어로 보고합니다.
   - **중요**: 보이스카우트 규칙에 따라 발견된 기존 코드의 결함도 개선 권고에 포함합니다.
5. **고급 검토**: 정밀한 AI 비평이 필요한 경우 `/jules` 워크플로우를 병행하여 심층 진단을 수행합니다.

## 통합 검토 체크리스트 (수동 파일 참조 금지)
에이전트는 검토 지시를 받으면 외부 `.md` 스킬 문서를 열람하지 말고, **오직 아래의 내장 체크리스트만을 기준**으로 즉시 분석을 수행합니다.

1. **클린 코드 & 구조**: 함수가 단일 책임(SRP)을 가지며, 네이밍이 명확한가? DTO와 Entity가 물리적으로 완벽히 분리되어 있는가?
2. **보안성**: SQL 인젝션, 무결성 위반이나 불충분한 인가(Authorization) 취약점이 없는가?
3. **런타임 및 예외 처리**: 예외를 단순히 삼키지 않고 커스텀 예외 체계를 따르는가? 비동기 레이스 컨디션이나 상태 불일치 위험은 없는가?
4. **리뷰 컨벤션**: 보이스카우트 규칙에 따라 기존 코드의 결함도 개선 권고에 포함했는가? (코드 작성 시 적용하는 Zero-Style Policy와는 별개로, 검토 시에는 구조적 리팩토링 및 런타임 개선점에 집중합니다)
