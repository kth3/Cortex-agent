# 정체성 인식 지능 (Identity Calibration)

에이전트는 사용자를 식별할 때 문자열 기반의 아이디에 매몰되지 않고, 실제 대화 맥락과 작업 데이터의 변화를 중심으로 사용자의 정체성을 동적으로 매핑해야 합니다.

## 1. 정체성 인식 대원칙 (Calibration Pillars)

1. **아이디 단정 금지**: Git Author(`user_id`, `user_name` 등)와 대화 중인 사용자의 관계를 성급하게 정의하지 않습니다. 한 사용자가 여러 계정이나 식별자를 사용할 수 있음을 항상 전제합니다.
2. **데이터 중심 분석**: "누가 고쳤는가"보다 "어떤 로직이 반영되었는가"에 집중합니다. 사용자가 특정 로직을 자신의 것이라고 명시할 경우, 그 즉시 해당 식별자를 사용자와 매핑합니다.
3. **가설 회피 (Confirmation Strategy)**: 인물 관계가 작업 방향이나 분석 결과에 영향을 줄 경우, 추측하지 말고 사용자에게 직접 확인합니다.

## 2. 작업 시 점검 사항 (Protocol)

- Git 로그나 Jira 이력을 분석할 때, 특정 팀원의 이름을 "나 이외의 타인"으로 자동 간주하지 않습니다.
- 과거 세션에서 저장된 `Identity Calibration` 지침이 있다면 최우선적으로 참조합니다.
- 사용자의 지시에 따라 언제든지 기존의 정체성 매핑을 수정할 준비를 합니다.

---

> [!TIP]
> 진정한 정체성 인지란 사용자의 이름을 외우는 것이 아니라, 사용자가 추구하는 **'로직의 방향성'**을 동기화하는 것입니다.
