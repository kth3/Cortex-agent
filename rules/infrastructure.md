# 인프라 무결성 지능 (Infrastructure Integrity)

에이전트는 자신의 도구(MCP, Scripts 등)가 프로젝트의 하위 저장소와 완벽히 동기화되어 있는지 지속적으로 확인하고, 결함 발생 시 비즈니스 코드 수정보다 인프라 수복을 최우선으로 합니다.

## 1. 인프라 우선 수복 (Maintenance-First)

1. **게으른 폴백 금지**: MCP 도구 에러 발생 시, 단순한 `grep`이나 기본 도구로 우회하여 분석의 정밀도를 낮추지 않습니다. 원인이 인프라(경로, 의존성, 스크립트 결함)라면 해당 인프라를 먼저 수복한 뒤 작업을 재개합니다.
2. **에러 가시화 (Infrastructure Diagnostics)**: 런타임에 파일이 반영되지 않거나 경로 인식이 어긋날 경우, `.agents/rules/diagnostics.md`의 프로토콜에 따라 환경 레이어를 먼저 점검합니다.
3. **Smart Root Discovery**: 프로젝트의 실제 루트(`.git` 등)를 동적으로 감지하여 작업 상대 경로의 정합성을 항상 유지합니다.

## 2. 지식 보고 정밀화 (Skill Reporting Calibration)

1. **실제 참조 기반**: 응답 하단의 `Skill` 항목은 인덱싱된 **1,200여 개의 실제 물리적 스킬 파일(`skills/**/*.md`)**을 참조했을 때만 해당 명칭(파일명)을 기재합니다.
2. **임의 명칭 대입 금지**: 작업의 성격이나 자의적인 요약 명칭(예: "system-mechanics-explanation")을 `Skill`에 넣는 것을 엄격히 금지합니다. 참조한 스킬이 없을 경우 `none`으로 표기합니다.

## 3. 고효율 백업 및 로드 (Symmetric Sync)

1. **압축 전송 원칙**: 대량의 미세 파일(`skills/`) 백업 시 반드시 `tar.gz` 압축을 통해 전송 효율을 극대화합니다.
2. **자동 복구 검증**: `/로드` 수행 시 압축 해제와 기존 데이터의 백업(Archive)이 전자동으로 이루어지는지 확인하여 인프라의 연속성을 보장합니다.

---

> [!IMPORTANT]
> 인프라는 에이전트의 '눈'과 '기구'입니다. 어두운 눈으로 코드를 수정하는 것은 금기사항입니다.
