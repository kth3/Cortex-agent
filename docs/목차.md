# .agents 사용설명서

---

## 1. 기본 사용법

### 아무것도 모르겠을 때
```
@코드
```
이것만 치면 에이전트가 현재 작업 파일을 보고 알아서 프로젝트 규칙과 관련 스킬을 참조합니다.

### 특정 스킬을 직접 부르고 싶을 때
```
@스킬이름 + 요청
```
예: `@brainstorming 이 기능의 설계를 논의하자`

스킬 이름만 쓰면 됩니다. 경로는 불필요합니다.

## 3. 사용하지 않아도 되는 스킬

아래 스킬들은 복사되어 있지만 현재 프로젝트(Java/Spring) 기준으로 **직접 사용할 일이 거의 없습니다**. 삭제할 필요는 없고(토큰 소모 없음), 참고 자료로 존재합니다.

| 스킬 | 이유 |
|---|---|
| `@senior-fullstack` | Node.js/React/Python 중심 풀스택 도구 (Java 미지원) |
| `@senior-architect` | Node.js/React 기반 아키텍처 도구 (위와 동일) |
| `@senior-frontend` | React/Next.js 전문가 역할 (필요하면 직접 사용 가능) |
| `@python-pro`, `@python-fastapi-development` | Python 프로젝트에서만 유의미 |

대신 사용할 스킬:
- Java 백엔드 → `@java-pro`
- 아키텍처 → `@architecture`, `@domain-driven-design`
- 풀스택 참고 → `@api-design-principles` + `@react-best-practices`

## 4. 핵심 @멘션 명령어

| @멘션 | 특징 | 사용 시점 |
|---|---|---|
| `@코드` | 규칙 로딩 + 맥락 기반 스킬 자동 참조 | 코드 작성 시작할 때 |
| `@작성` | 커밋/PR/Jira 문서 자동 생성 | 작업 완료 보고할 때 |
| `@검토` | 코드 품질 자가 점검 | 코드 완성 후 검토할 때 |
| `@동기화` | 브랜치 전환 시 메모리 스와핑 | Git pull/브랜치 변경 시 |
| `@테스트` | 테스트 실행 및 결과 분석 | 테스트 검증 시 |
| `@질문` | 코드 수정 없이 분석/계획만 | 안전하게 질문할 때 |
| `@분석` | 구조/의존성 시각화 | 코드 파악 시 |
| `@정리` | 불필요 코드 정리 | 코드 청소 시 |
| `@백업` / `@로드` | 구글 드라이브 백업/복원 | 설정 보존 시 |
| `@adr` | 기술 결정 기록 | 아키텍처 결정 시 |

**핵심**: 대부분의 작업은 `@진행`, `@코드`, `@작성`, `@검토` 4개만으로 충분합니다. 전문 스킬(`@java-pro` 등)은 더 깊은 지식이 필요할 때만 추가합니다.

## 6. rules/와 skills/와 workflows/ 차이

- **rules/ (자동 적용)**: 이 프로젝트만의 절대 규칙들. 파일 편집 시 자동 로딩됨.
- **skills/ (호출 시만)**: 범용 전문 지식. `@`로 호출하거나 rules에서 경로 참조하지 않으면 토큰 비소모.
- **workflows/ (멘션 호출)**: 프로젝트 특화 절차. `@진행`, `@작성` 등 한글 파일명으로 멘션하여 사용.
- **혼합 방식**: 자주 쓰는 핵심 스킬은 rules에 경로 참조로 자동 로딩, 나머지는 필요 시 `@` 호출.
